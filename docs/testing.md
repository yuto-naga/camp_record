# テストと品質管理 (Testing & QA)

本プロジェクトでは、「個人開発でも品質を落とさない」ことを目標に、自動テストの仕組みを導入しています。

## テストの種類

### 1. 単体テスト (Unit Test) - Vitest
ロジックの正確さを保証するために、関数や小さなコンポーネント単位でテストを行っています。
- **対象**:
  - `formatDate` や `cn` (クラス名結合) などの共通ユーティリティ関数
  - `StarRating` などのUIパーツのレンダリング
- **ツール**: [Vitest](https://vitest.dev/)
  - 極めて高速に動作し、開発中もリアルタイムにフィードバックを得られます。

### 2. E2Eテスト (End-to-End Test) - Playwright
ユーザーが実際にブラウザを操作する流れ（シナリオ）を自動化してテストします。
- **対象**:
  - 未ログイン時にログインページへリダイレクトされるか
  - ログインページが正しく表示されるか
- **ツール**: [Playwright](https://playwright.dev/)
  - PCやスマートフォンのブラウザ環境をエミュレートしてテストできます。

## CI/CD (GitHub Actions)

GitHubにコードをプッシュするたびに、以下の工程を自動実行するパイプラインを構築しました。
1. **環境構築**: Node.jsのセットアップ
2. **依存関係のインストール**: `npm install`
3. **単体テストの実行**: Vitestによる検証
4. **E2Eテストの実行**: Playwrightによるブラウザ動作検証

> [!TIP]
> **なぜ自動テストが必要か？**
> 修正を入れた際に「これまでは動いていた機能が壊れていないか（退行テスト）」を自動でチェックできるため、安心して機能追加を行うことができます。
